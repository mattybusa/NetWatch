WHAT WAS ATTEMPTED BUT FAILED
------------------------------

v3.3.19 arc — Jinja block nesting (self-introduced regression carried from v3.3.18):
During v3.3.18 preferences.html work, {% block scripts %} was inadvertently opened INSIDE
{% block content %}. In base.html these are separate sequential blocks. Jinja2 renders
the scripts content TWICE — once inline within content, and again at base.html's {% block
scripts %} location. Produced ~2023-line rendered HTML with duplicate APPEAR_KEY, causing
SyntaxError killing all JS. Fixed in v3.3.19 by separating blocks correctly.

v3.3.20 arc — CSS :root selector attempt:
Changed generate_css() to use :root selector. FAILED. CSS custom properties resolve by
DOM inheritance (nearest ancestor wins), not just specificity. body.theme-* sets variables
on body; body is closer to all elements than :root. Source order does not matter here —
body always wins over :root for CSS variable inheritance.

v3.3.19 — Fix: Jinja block nesting
- Closed {% block content %} before opening {% block scripts %} in preferences.html
- Removed extra {% endblock %} at end of file
- One APPEAR_KEY declaration in rendered output. JS restored. Confirmed working.

v3.3.20 — Fix attempt: :root selector (SUPERSEDED)
- generate_css() changed to use :root { ... }
- FAILED: CSS variable inheritance goes to nearest DOM ancestor (body, not :root)
- This package was accidentally uploaded to Pi via Deployed Files drag-and-drop
  (updater.py treats zip as raw file, copied it to ~/netwatch/). User deleted manually.

v3.3.21 — Fix: body[class] selector
- generate_css() changed to use body[class] { ... }
- body[class] has equal specificity to body.theme-* (element + attribute = element + class)
- Our <style> block is after themes.css in <head>, so we win on source order at tie
- body[class] always matches because body always has theme + layout classes present
- CONFIRMED WORKING: Neon Dystopia custom theme applied correctly across pages

NEW GOTCHAS ADDED
-----------------
(Appended directly to Gotchas.txt — this is a dev session)

1. JINJA2 NESTED BLOCKS CAUSE DOUBLE RENDERING (2026-03-13 | v3.3.19)
   {% block scripts %} nested inside {% block content %} causes double rendering
   if base.html defines them as separate sequential blocks. Produces duplicate JS
   declarations and silent SyntaxError killing all JS on the page.

2. CSS CUSTOM PROPERTIES: body BEATS :root FOR INHERITANCE (2026-03-13 | v3.3.19-v3.3.21)
   CSS variables resolve by DOM inheritance (nearest ancestor). body.theme-* wins over
   :root. To override: use body[class] placed after themes.css in <head>.

3. UPDATER.PY DRAG-AND-DROP TREATS ZIP AS A RAW FILE (2026-03-13 | v3.3.20)
   Uploading a .zip via Deployed Files copies it as-is to ~/netwatch/ — does NOT extract.
   Package Installer (patcher.py) is the correct path for zips.

4. DON'T CHASE CACHE — IT'S ALWAYS THE CODE (2026-03-13 | v3.3.21)
   In NetWatch's history, unexpected post-restart behavior has ALWAYS been a code issue,
   not browser or gunicorn cache. Check rendered HTML source first.
