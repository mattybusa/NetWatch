v3.2.98 — North Star statement added to AI_Session_Process.txt. Full text:
  "NetWatch is a self-hosted home network monitoring and automation platform running on
  a Raspberry Pi. It monitors network health, automates recovery, and gives homeowners
  visibility and control they wouldn't otherwise have. The current focus is the internet
  connection and the equipment behind it — but the platform is designed to grow. The
  long-term vision is full home infrastructure monitoring: connected devices, network
  interfaces, telemetry, and intelligent alerting, all managed through a single interface

AI_Session_Process.txt bumped to v1.2.

v3.2.99 — Patch manager page now reloads after all install outcomes. Previously the
monitor-only and no-restart cases left the page in an incomplete-looking state requiring
a manual refresh. Fixed by adding an else branch (no restart) and modifying the monitor
branch to reload after confirmation. All three cases now end with a page reload:
- Web restart: polls until server back, then reloads (unchanged)
- Monitor restart: shows confirmation, reloads after 3s delay
- No restart: shows "✓ Package applied — reloading..." after 1.5s, reloads after 1s
Confirmed working via v3.2.99.1 test package (doc-only, no restart).

DECISIONS MADE
--------------
- Database schema and config keys: no separate reference doc needed. The source files
  (database.py, auth.py, alert_subscribers.py, patcher.py, security_log.py,
  config_validator.py, config_schema.py) are self-documenting with inline comments.
  Pointers added to context doc are sufficient. Copying schemas to dev docs would
  create a maintenance drift problem.

- Two new process rules added to AI_Session_Process.txt "During a Session" section:
  1. Inline documentation standards must be maintained — new functions, tables, and
     non-obvious logic get comments matching the existing file style.
  2. Context doc must be kept current — when adding functions to modules referenced
     by name in the context doc, update the doc entry in the same package.

- North Star statement drafted collaboratively. User confirmed Option C from three
  drafts, with the final line revised from "built to be given to a friend" to hint
  at a wider future without overstating current plans.

- Patch manager reload: made consistent across all three outcomes rather than just
  fixing the no-restart case. Monitor-only was previously show-and-stay; now
  show-and-reload for consistency.

- Session kept intentionally short to test the new session continuity infrastructure
  with a clean closeout.
