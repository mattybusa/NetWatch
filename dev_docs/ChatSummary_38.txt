WHAT WAS ACCOMPLISHED
---------------------

PHASE 1 — Security Audit

Full code audit across patcher.py, webapp.py, auth.py, config_schema.py, and all
upload/restore routes. Findings in four tiers:

CRITICAL (must fix before public promotion):
  C1 — No upload size limit. OOM risk on Pi 3B+. Fix: MAX_CONTENT_LENGTH = 10MB.
  C2 — DB restore accepts any SQLite file with only a single table check. Crafted DB
       could inject a backdoor admin account. [PENDING]
  C3 — run_sql manifest action executes arbitrary SQL with no restrictions.
       Could DROP tables, INSERT backdoor accounts, DELETE all data. [PENDING]
  C4 — No package signature verification. Any well-formed zip installs. [PENDING]

HIGH (fix before wider distribution):
  H1 — No HTTP security headers (X-Frame-Options, X-Content-Type-Options, CSP).
  H2 — Session cookie not rotated after MFA. Pre-MFA captured cookie stays valid.
  H3 — Reset tokens valid simultaneously with real password (deliberate — documented).
  H4 — Git PAT in patch_log.git_push_error on push failure. [PENDING, low urgency]

MEDIUM (harden when practical):
  M1 — Deployed files uploader: file.filename not sanitized. Path traversal risk.
  M2 — Dev docs uploader: same as M1.
  M3 — setup/skip: no post-setup guard. Can rotate SECRET_KEY accidentally.
  M4 — Config export returns credentials in JSON (admin-only + HTTPS mitigates).
  M5 — Brute-force protection user-scoped only, no per-IP rate limiting.

ACCEPTED/LOW (no action needed):
  A1 — config.py plaintext credentials: deliberate LAN trade-off.
  A2 — Hand-rolled auth: deliberate RAM constraint trade-off.
  A3 — Self-signed TLS: correct for LAN deployment.
  A4 — netwatch-svc PAT in git remote: low risk on home LAN.
  A5 — Snapshots exclude config.py: correct and intentional.


PHASE 2 — Implementation (v3.4.41–v3.4.43)

v3.4.41 — Five fixes to webapp.py, web restart only:

  C1 — MAX_CONTENT_LENGTH = 10 * 1024 * 1024 (10MB)
    Flask enforces at HTTP layer before file.read(). Covers all upload routes.
    10MB: generous ceiling for any realistic NetWatch file with 2x headroom.

  H1 — HTTP Security Headers
    Renamed after_request handler set_cache_headers -> set_security_headers.
    Added to ALL responses:
      X-Frame-Options: DENY — prevents clickjacking
      X-Content-Type-Options: nosniff — prevents MIME sniffing
      Content-Security-Policy — allowlists 'self' + cdn.jsdelivr.net
        (Chart.js and Bootstrap Icons are the only external CDN resources).
        'unsafe-inline' required for script-src/style-src: Flask embeds server-side
        data in inline <script> blocks throughout templates. Documented limitation:
        inline XSS not blocked, but external script loads and data exfiltration are.

  M1 — secure_filename in api_update_apply (Deployed Files upload)
    werkzeug.utils.secure_filename applied to file.filename. Empty string check added.

  M2 — secure_filename in api_devdocs_upload (Dev Docs upload)
    Same pattern as M1.

  M3 — setup/skip post-setup guard
    is_first_run() check at top of route. If setup complete, redirects to index
    without touching config.py. Prevents accidental SECRET_KEY rotation post-install.

v3.4.42 — Unplanned fix discovered during v3.4.41 testing:
  Flask 413 response is plain HTML. Package installer JS called res.json() on it,
  threw silently, left "Validating file..." spinner running forever. Fixed both
  preview and apply fetch blocks in patch_manager.html:
    - res.status === 413 -> "File is too large. Maximum upload size is 10 MB."
    - Other non-OK status -> "Server error (N) — check logs."
    - try/catch around fetch -> "Request failed — check network connection."

v3.4.43 — H2: Session rotation post-MFA:
  session.clear() added to _complete_mfa_login() before auth.login_user().
  next_url and must_change saved to locals before clear, used after login_user().
  Any cookie captured between password entry and MFA completion becomes worthless.
  Redundant explicit session.pop() calls removed (session.clear() handles them).

DECISIONS MADE
--------------
- 10MB upload limit: covers realistic files, meaningfully restricts abuse.
- CSP 'unsafe-inline': required by Flask inline data pattern. Removing requires
  Blueprint refactor. Documented in code comments — not a mistake.
- cdn.jsdelivr.net only external CDN (Chart.js + Bootstrap Icons, no Google Fonts).
- setup/skip redirects to index (not login): user likely already logged in.
- H2 session.clear() is safe: all pre-MFA keys are transient and discarded anyway.
  next_url and must_change saved to locals before clear, restored after.
- C2, C3, C4 deferred: higher implementation risk, need more design thought.
- H4 deferred: low urgency.
- Publish to GitHub held: wait until C2 + C3 ship for a cleaner security bundle.
