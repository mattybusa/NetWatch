WHAT WAS ATTEMPTED BUT FAILED
------------------------------
- Initial latency query without aggregation wrapper returned No data. Root
  cause: unwrap requires a range aggregation function (avg_over_time) for
  Loki to produce a metric series that Grafana can plot. Fixed by wrapping
  in avg_over_time([1m]) then collapsing multiple series with outer avg().

- Initial latency query produced dozens of legend entries labeled "Latency"
  instead of one. Root cause: Loki returns one series per unique label set.
  Fixed by wrapping in avg() to collapse all series into a single line.


DECISIONS MADE
--------------
- Design brief first: field names verified via Explore before building any
  panels. Correct approach — the unwrap field names must match exactly.

- avg(avg_over_time([1m])) pattern for metric panels: inner function buckets
  by 1-minute window, outer avg collapses multiple Loki streams into one line.
  This is the correct pattern for NetWatch metric panels going forward.

- Logs panel type for Reset Events: raw structured lines are more useful than
  a chart for reset events — timestamp and reset_type visible at a glance.

- No value = 0 for Errors & Warnings: displays 0 instead of "No data" on a
  clean system, which is more informative for a health metric.

- Dashboard-level time range selector is the primary interaction model. Per-
  panel time overrides are static design-time settings, not dynamic user
  controls. Dashboard selector is sufficient for operational use.

- Panel count ended at 7 (not 6 as originally planned) — Monitor Last Seen
  and Web Last Seen are two separate stat panels.

- End-of-session context doc updated at v3.3.38 (no version bump — no
  NetWatch code changes this session).
