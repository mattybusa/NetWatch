v3.3.77 — Compact layout fix (index.html, web restart):
  User reported the gap was too large in compact layout. Root cause: compact layout

  hardcoded at 8px — too large for compact.
  Fix: changed padding-top:8px to padding-top:calc(var(--gap,16px)*0.5), matching the
  exact formula used by .row>* in base.html. Correct in all layouts:
    Comfortable/Dashboard-first/Sidebar: 16px * 0.5 = 8px
    Compact: 10px * 0.5 = 5px
  Confirmed working by user.


WHAT WAS ATTEMPTED BUT FAILED
------------------------------
Six consecutive wrong packages (v3.3.70 through v3.3.75) trying to fix row gap:

The core misunderstanding: multiple theories about what was creating row 1->2 gap were
wrong. The real cause was never the .row margin-bottom or mb-4 class — it was the
padding on the .row>* column wrappers (8px on all sides). The flex div's children have
no col wrappers, so they have zero padding on top. Browser measurement was the only
way to confirm this.

Key wrong theories tried:
- margin-top:0 removed all gap (zero, not equal)
- margin-top:1.5rem doubled the gap (mb-4 was also adding spacing — except it wasn't,
  because .row rule overrides mb-4 in source order)
- removing mb-4 from uptime row eliminated gap entirely
- margin-top:-8px and calc variants had no effect because the row's negative margin
  was already accounted for differently than expected

Root cause of the confusion: in base.html's inline <style> block, .row is defined
AFTER .mb-4 (line 59 vs line 49). CSS source order means .row's margin shorthand
(-8px all sides) overrides mb-4's margin-bottom:20px on any element with both classes.
So mb-4 has never worked on .row elements. The apparent row 1->2 gap was never from
mb-4 — it was always from col wrapper padding.


DECISIONS MADE
--------------
- padding-top:calc(var(--gap,16px)*0.5) is the correct and permanent formula for the
  flex row container. It mirrors .row>* exactly and auto-adjusts per layout. Do not
  change to a hardcoded value or margin-based approach.

- Browser measurement (getBoundingClientRect) via Claude in Chrome is the right tool
  when CSS spacing behavior is ambiguous. Don't ship theories — measure first.

- New backlog item added: package installer confirm row spacing (the "Confirm" step row
  in the install progress UI touches the box below it).
