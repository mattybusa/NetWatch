ARC: Dev UI Hiding (v3.4.20)
-----------------------------
BACKGROUND
Deployed Files page and Download Current Code button are development-only tools
that serve no purpose on standard user installs. The Release Manager was already
gated by DEVELOPMENT_SYSTEM. Extending that pattern to two more admin page elements.

WHAT WORKED
is_dev_system is already injected into every template via the global context processor
in webapp.py — no Python changes needed, only admin.html.

Changes to admin.html:
- Deployed Files card: wrapped entire card in {% if is_dev_system %}...{% endif %}
- Export Data card: removed the hr separator and Download Current Code button + its
  description paragraph. Card now contains only the two CSV export buttons.
- New Developer Tools card: added after Export Data, gated by {% if is_dev_system %},
  contains the Download Current Code button and its description. Uses section ID
  "devtools" and chevron ID "chevron-devtools" to avoid conflicts.

Decision: hide the entire Deployed Files page (not just the Dev Docs section within
it). Users who need single-file replacement on a non-dev install can SSH+SCP or
temporarily set DEVELOPMENT_SYSTEM=True.

Confirmed: test Pi (non-dev) shows neither card. Production Pi (dev) shows both.


ARC: Release Manager Layout (v3.4.22–v3.4.23)
-----------------------------------------------
BACKGROUND
After the Generate button and hint div were added in v3.4.21, two layout issues
appeared: (1) clicking Generate caused the row to shift upward as the hint div
appeared, and (2) after that fix, the version input and Build button were permanently
misaligned with the description column.

WHAT WAS TRIED AND FAILED
v3.4.22 first attempt: switched hint div from display:none to visibility:hidden so
it always occupies space. This fixed the shift-on-generate but created a permanent
height difference — the hint div added ~1em below the description column that the
other two columns didn't have, pushing version input and Build button up relative
to the description input.

WHAT WORKED
Root cause: outer row uses align-items:flex-end. Description column is taller due
to the always-present hint div. Other columns align to the bottom of that taller
column, making them appear higher.

Fix (v3.4.23): added padding-bottom:calc(1em + 3px) to the version column div and
wrapped the Build button in a div with the same padding-bottom. All three columns
now have identical floor height. Row stays level whether hint is visible or not.


ARC: Release Description Bullets (v3.4.24, v3.4.27)
------------------------------------------------------
BACKGROUND
Release description generated by the Generate button was not rendering as a bullet
list on GitHub. Entries were running together in one paragraph.

WHAT WAS TRIED AND FAILED
v3.4.24 added "- " prefix to each entry and used "\n".join(entries). This was
correct for Markdown but the newlines were not surviving to GitHub. Root cause:
description field was a single-line <input type="text"> which strips newline
characters from its .value. The payload sent to publish_release had \n replaced
with spaces or dropped entirely.

WHAT WORKED
Two-part fix:
- v3.4.24: entries prefixed with "- " in admin_release_notes_draft route
- v3.4.27: description <input type="text"> replaced with <textarea rows="3"
  style="resize:vertical"> in admin.html. Textarea .value preserves \n characters.
  .trim() used in JS only strips leading/trailing whitespace, not internal newlines.

Both changes required. Confirmed working after publish — GitHub shows bullet list.
