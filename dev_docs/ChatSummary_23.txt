v3.3.61 — Gunicorn lifecycle JSON fix (web restart only):
  Root cause of v3.3.60 gunicorn logging failure: _emit() called
  structlog.get_logger() but structlog wasn't configured yet at startup time.
  gunicorn.conf.py loads before wsgi.py runs _configure_structlog().
  Fix: _emit() now writes a JSON line directly to stdout using print(flush=True)
  instead of using structlog. stdout is always captured by journalctl regardless
  of logging configuration. JSON schema matches structlog output exactly:
  {service, event, message, level, timestamp}.
  The debug() override also captures gunicorn's internal request debug logs
  (separate from the access() HTTP log entries) — slightly noisy but harmless.
  Confirmed working: all lifecycle messages (shutdown, worker exits, startup,
  Listening at, Booting worker) visible as JSON in journalctl.
  Session 23 badge confirmed in v3.3.61 release notes. ✅


WHAT WAS ATTEMPTED BUT FAILED
------------------------------
- v3.3.60 gunicorn lifecycle logging: used structlog.get_logger() in _emit().
  Structlog unconfigured at gunicorn startup → messages silently dropped.
  No journalctl output for the startup window after v3.3.60 install confirmed this.
  Fixed in v3.3.61 by switching to direct stdout print.

- Session number in v3.3.60 release notes: the old in-memory patcher (preload_app=True)
  ran the install, so _log_patch() was the old version without the session parameter.
  Row was written with session=NULL. Expected behavior, not a bug.
  Confirmed via: sqlite3 netwatch.db "SELECT package_version, session FROM patch_log
  ORDER BY timestamp DESC LIMIT 5;" → 3.3.60 row had empty session.


DECISIONS MADE
--------------
- Third-party system services (SSH, systemd, OS, Node Exporter, Promtail) sending
  plain text to Loki is acceptable and expected. Only NetWatch-controlled code is
  expected to emit JSON. No OTEL Collector or pipeline stages needed for these.

- gunicorn lifecycle messages: use direct stdout print rather than structlog, because
  gunicorn.conf.py loads before wsgi.py configures structlog. This is a permanent
  architectural constraint, not a workaround.

- debug() override left in place: captures gunicorn's internal request debug logs.
  Slightly noisy but acceptable. Could add a level guard if desired in future.

- System log viewer scoped to pkg_update.log only (Approach A). All other logs are
  already covered by the Logs tab or Grafana. pkg_update.log was the only gap.

- Session field in manifest: optional, string-coerced. Convention going forward:
  always include "session": N in every manifest.json. Not enforced by patcher.

- SystemSetup.txt is append-and-update (not append-only like Gotchas.txt). Updated
  whenever manual steps are performed or gaps are closed.
