DECISIONS MADE
--------------
- config_schema/configeditor consolidation approach: add type, section, min, max,
  ui_hidden to each config_schema.py entry; configeditor.py builds FIELDS dynamically.
  group (logical category for validator) and section (display heading for UI) kept as
  separate fields — they serve different purposes and don't map 1:1.

- ui_hidden=True for SECRET_KEY: it is validator-managed but should never be editable
  via the UI. Any future key that should be validator-only gets the same treatment.

- Dated context doc backups: user confirmed they will manually delete aged-out files.
