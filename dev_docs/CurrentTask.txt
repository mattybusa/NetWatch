KNOWN BUG: theme_manager.py PERMISSION ERROR
---------------------------------------------

(the service account home directory). This causes a PermissionError on every admin
page load and means themes are broken. The expanduser() fix applied to main.py and
theme_manager.py in v3.3.25 did not fully resolve this -- theme_manager.py still
has at least one path that resolves incorrectly under the netwatch-svc account.

Symptom: ERROR in app on /api/themes -- PermissionError: /nonexistent
Impact: Themes fail to load. Non-critical but visible error on every admin load.
Fix: Audit theme_manager.py for any remaining expanduser() or hardcoded home
     directory references. Replace with os.path.dirname(os.path.abspath(__file__))
     pattern. Address in audit session or as first code task after audit.
