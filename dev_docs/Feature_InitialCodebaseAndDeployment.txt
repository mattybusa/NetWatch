WHAT WAS TRIED AND FAILED
--------------------------
First generation attempt (lost to context compaction):
The initial code generation attempt produced files that were then lost when the context
was compacted. The user's opening message in the 09:00 transcript is "not again, we have
already done that, we were going in circles and then you finally generated the code, now
it's gone." This confirms at least one prior generation was completed and lost.

Second generation — consolidated structure (rejected by user):
After the first loss, a second attempt produced a consolidated (single-file or
non-modular) structure rather than the original modular design. The user reviewed the
options and selected "Rebuild with the original modular structure (recommended)."

File download/paste failure:
Early attempts to provide files via code blocks for copy-paste to Pi via nano had mixed
results. Some files downloaded incorrectly (missing comments). The user correctly noted:
"so i was able to download that one and it doesn't look the same, it is missing all the
notes. if they aren't going to be the same I would prefer to have the actual files."
The solution was to provide full files with complete comments preserved.

GPIO edge detection error on first service start:
After deployment, netwatch-monitor failed to start. journalctl showed: "Failed to add
edge detection" on the button GPIO pin. Root cause: GPIO.setup() for the button pin was
being called after relay.py had already initialized GPIO in a conflicting state. Fixed by
reordering GPIO initialization in the service startup sequence / button.py.

DEPLOYMENT WORKFLOW:
1. Files packaged as netwatch_v2.zip
2. SCP transfer from Windows to Pi:
   scp netwatch_v2.zip mboynton@192.168.100.10:~/
   (Note: must be in a separate PowerShell window, not inside the SSH session)
3. SSH to Pi, unzip:
   cd ~ && unzip netwatch_v2.zip
4. Install Python dependencies in venv:
   cd ~/netwatch && python3 -m venv venv && source venv/bin/activate
   pip install flask sqlalchemy speedtest-cli RPi.GPIO
5. Configure: nano ~/netwatch/config.py (set IPs, credentials, keys)
6. Install systemd services:
   sudo cp netwatch-monitor.service /etc/systemd/system/
   sudo cp netwatch-web.service /etc/systemd/system/
   sudo systemctl daemon-reload
   sudo systemctl enable netwatch-monitor netwatch-web
   sudo systemctl start netwatch-monitor netwatch-web
7. Verify: sudo journalctl -fu netwatch-monitor
   (Should show check results every 30 seconds)
