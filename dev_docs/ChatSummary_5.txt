WHAT WAS ATTEMPTED BUT FAILED
------------------------------
- v3.3.3 rollback.sh fix (superseded): heredoc replaced with printf-built temp file —
  the correct fix for DB restore — but the file still contained Unicode box-drawing
  characters and arrows from the original script. Bash threw a syntax error on the Pi
  at runtime. Superseded immediately by v3.3.4 which rewrote the entire script in
  pure ASCII.

- Rollback attempt during v3.3.2 incident: web service went down after v3.3.2 install.
  Matt attempted a rollback to v3.2.99.2. Script ran without apparent error but the DB
  restore did not execute. Root cause: original rollback.sh used a bash heredoc
  (python3 - << PYEOF) which failed silently on the Pi. Version remained at v3.3.2
  after rollback. Web service recovered manually: sudo systemctl start netwatch-web.service
  DB restore was fixed in v3.3.4.

- v3.3.0 About popover initial UI: color rendered incorrectly (button inherited wrong
  color) and popover was too narrow for the dependency table. Fixed in v3.3.1.

NEW GOTCHAS ADDED
-----------------
Gotchas were appended directly to Gotchas.txt during this dev session per updated process.
No separate Gotchas_Append_Session5.txt produced. Entries added:

- PURE ASCII ONLY IN ALL .sh FILES (new) — Unicode in shell scripts causes bash syntax
  errors on the Pi when transferred via SCP or Dev Docs upload, even in comments
- NEVER USE BASH HEREDOCS IN SHELL SCRIPTS (new) — bash heredocs (<<) fail silently
  on the Pi; use printf to build temp files and pass as sys.argv instead
- ROLLBACK RESTORES .sh FILES FROM SNAPSHOT (new) — installing a fixed rollback.sh
  then rolling back will restore the old broken version from the snapshot
