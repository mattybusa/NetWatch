WHAT WAS ATTEMPTED BUT FAILED
------------------------------
- First approach: moved netwatch directory to /home/netwatch-svc/netwatch and updated
  all shell scripts to use absolute /home/netwatch-svc/ paths. This was wrong —
  it created a home directory requirement that undermines the security benefit.
  A netwatch-svc home dir was created and then deleted (sudo userdel -r netwatch-svc).
  First package (netwatch_v3.3.25.zip, labeled v3.3.25) was discarded entirely.
  User caught the error at Step 3 of the manual install instructions.

- Second package (v3.3.25b zip): correct approach, but patcher.py failed during install
  with "syntax error in main.py" — misleading error caused by a permissions problem on
  /tmp/__pycache__/ (world-writable dir set to wrong owner). Fixed with
  sudo chmod 777 /tmp/__pycache__, then install succeeded.

- After package install, netwatch-monitor crashed immediately with:
  PermissionError: [Errno 13] Permission denied: '/nonexistent'
  Root cause: the package had not yet been installed when the monitor was first
  restarted — it was still running the old main.py with expanduser(), which resolved
  ~/netwatch/logs/ to /nonexistent/netwatch/logs/ for netwatch-svc. Resolved by
  stopping the monitor and installing the package first.

- netwatch-web failed with status=200/CHDIR (Permission denied). Root cause:
  /home/mboynton was 700 (owner-only). netwatch-svc could not traverse the parent
  directory to reach WorkingDirectory=/home/mboynton/netwatch. Fixed with
  sudo chmod 751 /home/mboynton.


DECISIONS MADE
--------------
- netwatch-svc created with --no-create-home --home-dir /nonexistent. No home
  directory. Files stay at /home/mboynton/netwatch/ owned by netwatch-svc.
  This was the correct approach and what was originally planned. The first build
  drifted from it and had to be corrected.

- Shell scripts (backup.sh, rollback.sh, etc.) were NOT changed. They run as mboynton
  over SSH, so $HOME correctly resolves to /home/mboynton. No change needed.

- /home/mboynton chmod'd to 751 (not 755). Execute-only for others: netwatch-svc can
  traverse into it but cannot list its contents. Deliberate tradeoff — slightly less
  locked than 700 but required for service to work without moving the netwatch dir.

- /tmp/__pycache__ cleanup: deleted entirely after confirming it was 11MB of patcher
  temp .pyc files accumulated since March 2, never cleaned up. Patcher bug — backlog
  item added.

- Context doc updated in-package at v3.3.25. Both backlog corrections and service
  account completion recorded there.
