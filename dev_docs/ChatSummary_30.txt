  2. Created Gitea PAT "netwatch-pi" with repository read+write, no expiration.

  3. Updated remote URL to ops-pi.home with PAT embedded:
       git -C ~/netwatch remote set-url origin \
         https://mboynton:<PAT>@ops-pi.home:3000/mboynton/netwatch.git

         user.name "NetWatch Pi"

v3.3.91 — GIT INTEGRATION (patcher.py, webapp.py, patch_manager.html):
  patcher.py:
  - _is_git_configured(): checks for .git dir and remote origin; returns False
    if git not set up (enables graceful skip on distributable installs).
  - _git_push(version, session, description): stages all, commits with message
    "vX.X.X (Session N): description", pushes to origin main. Handles timeout
    (60s for push), "nothing to commit" treated as success, errors returned cleanly.
  - _save_git_state(patch_log_id, git_result, ...): persists push result to
    patch_log table (git_push, git_push_error, git_version, git_session,
    git_description columns — added via ALTER TABLE migration).
  - get_last_git_state(): returns most recent install's git push state for UI.
  - retry_git_push(): reruns push for most recent install without reinstalling.
  - get_git_configured(): public wrapper for webapp.py UI decisions.
  - _ensure_patch_log_table(): updated with 5 new git columns (safe ALTER TABLE).
  - apply_package(): calls _git_push() after successful install; stores result;
    returns git_push and git_push_error in response dict.

  webapp.py:
  - GET /api/patch/git_status: returns {configured, last} for UI polling.
  - POST /api/patch/git_retry: triggers retry_git_push(), returns result.

  patch_manager.html:
  - New git push result card below install result panel (hidden when git not configured).
  - Shows ✓ Pushed to Gitea (green) or ✗ Push failed with error detail (red).
  - Retry button appears only on failure — calls /api/patch/git_retry, polls result.
  - pollGitStatus() called after successful install; polls twice with delay to
    allow async push to complete before reading DB state.
  - resetUI() clears git row on "Install Another Package".

  NOTE: v3.3.91 itself showed no git push row — expected. preload_app=True in
  gunicorn means the old in-memory patcher ran the install. From v3.3.92 onward
  every install will auto-push and show the row.


WHAT WAS ATTEMPTED BUT FAILED
------------------------------
- "sudo -u netwatch-svc git -C ~/netwatch config ..." failed with "fatal: not in
  a git directory". Root cause: tilde expands to /nonexistent for netwatch-svc.
  Fixed by using full absolute path /home/mboynton/netwatch.

- "sudo -u netwatch-svc git config --global ..." failed with "could not lock config
  file /nonexistent/.gitconfig". Root cause: netwatch-svc has no home directory.
  Fixed by using repo-local config (no --global flag, full path via -C).

- .gnupg/ and pkg_update.log committed in initial push before .gitignore was updated.
  Cleaned up in a follow-on commit. No security risk (.gnupg contains only public
  key material; pkg_update.log contains no secrets).


DECISIONS MADE
--------------
- Every package commits — completeness over cleanliness. Future Matt will appreciate
  the complete history.
- dev_docs in git. AI session files are documentation, not secrets. Full history
  is useful for mBase extraction.
- No Gitea Issues automation. Backlog in context doc is sufficient; split tracking
  introduces sync risk with minimal gain.
- No session workflow automation. Zip upload is reliable; filter problems from
  Sessions 11-13 not worth revisiting without a fundamentally different approach.
- PAT embedded in remote URL as the credential mechanism. Simpler than credential
  store for a service account with no home directory.
- Kiosk display permanently descoped from NetWatch.
- Git commit format: "vX.X.X (Session N): description" — pulls all three fields
  from manifest.json which patcher already reads.
- git_push failure is non-blocking — install success is independent of push success.
