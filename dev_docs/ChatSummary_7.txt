  pkg_update.sh: new shell script (pure ASCII, no heredocs). Runs apt-get
  update then apt-get upgrade --dry-run to capture package list, then
  apt-get upgrade -y. Passes raw dry-run output to pkg_update_alert.py for
  parsing and alert firing. Logs all activity to pkg_update.log. Supports
  --alert and --quiet flags for cron use.

  pkg_update_alert.py: new Python helper invoked by pkg_update.sh. Three
  modes: LOG (parse dry-run output, print package list to stdout for log
  capture), OK (fire pkg_update alert with summary + package detail), FAILED
  (fire pkg_update alert with failure message). parse_dry_run() extracts
  package name, old version, new version from apt Inst lines using regex.
  Alert message includes full per-package version delta list.

  Sudoers entry (manual, one-time): three lines in /etc/sudoers.d/netwatch-apt
  covering apt-get update, apt-get upgrade -y, and apt-get upgrade --dry-run.
  chmod 440 required. Must be done before enabling the schedule.

  Cron verification command: crontab -l | grep netwatch-pkg-update

  (see Feature_PkgUpdateScheduling.txt)


WHAT WAS ATTEMPTED BUT FAILED
------------------------------
- pkg_update.sh first draft used python3 - << PYEOF heredoc for dry-run
  parsing inline in the shell script. Caught and removed before shipping —
  NEVER USE BASH HEREDOCS rule applies. Replaced with pkg_update_alert.py
  helper invoked via python3 path args instead.

- pkg_update.sh second draft used a while-read loop fed by a heredoc on a
  variable (<<DRYEOF). Technically different from the python3 heredoc but
  still a heredoc pattern on the Pi. Removed in favour of writing dry-run
  output to a temp file and passing the file path to the Python helper.


DECISIONS MADE
--------------
- apt-get upgrade -y (not dist-upgrade): conservative choice for a live
  production Pi. Will not remove packages or pull in unexpected new
  dependencies. Kernel updates are included and require manual reboot to
  take effect — this is acceptable behaviour.

- Narrow sudoers entry: three specific apt-get commands only, not blanket
  NOPASSWD: ALL. Appropriate for home network Pi on trusted LAN.

- Service account (netwatch-svc) deferred to backlog: the narrow sudoers
  entry is the important protection. Service account refactor is a full
  session of its own and does not belong bolted onto this feature.

- All Python logic in pkg_update_alert.py, not inline in shell: keeps the
  shell script free of heredocs and avoids multi-line quoting problems.

- pkg_update.log is disk-only with no UI viewer: alert email covers the
  normal case. Log viewer added to backlog as a future UI item.

- Label for schedule alert toggle: "Send alert on completion" rather than
  "Email" — distinguishes it clearly from the backup email toggle which
  sends a file attachment.

- Daily backup Enabled UI fix: changed from dropdown (Yes/No) to checkbox
  toggle in the same package, since we were already touching admin.html.
  Consistency fix, no functional change.

- Two new backlog items added:
  UI: System log viewer (pkg_update.log tail in Admin page)
  Infrastructure: Dedicated service account (netwatch-svc)

- End-of-session context doc update at v3.3.12. Backlog version tag updated,
