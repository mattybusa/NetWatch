BUGS FOUND AND FIXED DURING TEST PLAN
--------------------------------------

1. get_user_by_id missing role JOIN — after MFA challenge passed, login_user() received
   a dict with no role_name, permissions, or session_minutes. Session would have been
   broken. Fixed by adding JOIN roles to the SELECT.

2. mfa_setup error path regenerated secret — entering wrong TOTP code during setup
   called setup_mfa() again, generating a new secret and overwriting the one already
   scanned. Fixed to use get_mfa_setup_uri_from_secret() (read-only).

3. mfa_send_code was GET-only — template forms POST to /mfa/send_code but route only
   accepted GET. Fixed to accept GET and POST, reading channel from form or args.

4. Missing alerts import in mfa_send_code — NameError: name 'alerts' is not defined.
   Fixed by adding local import at top of function.

5. Password verification check inverted — both mfa_disable and mfa_regenerate_backup_codes
   used isinstance(verified, dict) to detect failure, but authenticate() always returns
   a dict on success. Fixed to not verified.get("id").

6. QR code generation required Pillow — qrcode.make_image() defaults to PIL renderer.
   Pillow not installed. Fixed by switching to qrcode.image.svg.SvgPathImage (pure
   Python, ships with qrcode package, no extra install).

7. Stray backslashes in preferences.html section comment headers — pre-existing issue,
   fixed while editing the file.

8. MFA card in wrong position — was after Dashboard Preferences. Moved to immediately
   after Change Password card.

9. Banner dismiss not persisting across navigation — client-side only hide didn't
   survive page loads. Fixed with server-side session flag via /api/mfa/dismiss_banner.

10. Regen modal copy button rendered JS as text — inline onclick with nested quotes
    broke HTML attribute. Fixed by using named copyRegenCodes() function and module-level
    _regenCodes array.

11. Regen modal closeable after codes shown — backdrop click dismissed modal before
    user saved codes. Fixed: modal locked after codes shown, Done button only enabled
    after copy clicked, cancel button hidden.

12. Literal newline in JS string — _regenCodes.join('\n') written with real newline,
    causing JS parse error that killed entire script block. Fixed to \\n.

13. Backup count not refreshing after regen — closeRegenModalForce() only hid modal.
    Fixed to reload page.

14. Re-configure MFA overwrote active secret — GET handler called setup_mfa() which
    wrote new secret to DB immediately, before confirmation. Cancelling destroyed the
    working secret. Fixed: setup_mfa() no longer writes to DB. Pending secret stored
    in session["mfa_setup_secret"] only. confirm_mfa_setup() writes to DB after
    verification. Cancel clears session secret, DB untouched.

15. confirm_mfa_setup webapp routes not updated — string replacements silently failed
    in an earlier fix attempt. Routes still called old 2-arg confirm_mfa_setup(). Fixed
    by rewriting both mfa_setup and mfa_setup_confirm routes cleanly.


DECISIONS MADE
--------------
- SVG QR codes instead of PNG — no Pillow dependency, sharper rendering, cleaner.
- Pending MFA secret lives in Flask session only until confirmed — never written to
  DB on GET. This is the correct pattern for any setup flow where cancelling must
  be safe.
- Grace period banner: dismissible during grace (reappears each new login), not
  dismissible when expired. Implemented server-side via session flag.
- MFA card position: immediately after Change Password in My Account.
