v3.4.34 — Release Manager: lock fields after build, Reset button, description
  in zip manifest. After Build Release, all inputs (version, description, generate,

  POST /api/admin/discard_release to delete the zip, then re-enables everything.

  written to temp file and passed to Python manifest block. Release zip manifest.json

  doPublishRelease() reads from _releaseData.description (locked at build time).

v3.4.35 — MFA disable toggle. New MFA_REQUIRED bool config key (default True,
  Security section). mfa_enforcement_enabled() helper in auth.py reads it at runtime.
  Three enforcement points in webapp.py gated: before_request grace banner, login
  route forced-setup redirect, per-user disable grace deadline restart. Security card
  added to Admin page (above Backup Manager) with toggle button, description, and
  orange warning strip when disabled. Toggle POSTs to /api/config/save and reloads
  after restart. Both services restart (auth.py change). Validated all test steps.

v3.4.36 — Configurable summary frequency. Three new config keys: SUMMARY_FREQUENCY
  (daily/weekly, default daily), SUMMARY_DAY (0-6, default 0=Monday), SUMMARY_HOUR
  (0-23, default 8). main.py scheduling block reads all three at runtime each loop
  iteration — no restart needed after config change except monitor restart.
  New-keys modal fires on next Config Editor open.

v3.4.37 — Config Editor/modal improvements (first attempt). Added allowed_values to
  SUMMARY_FREQUENCY in config_schema.py. configeditor.py passes allowed_values and
  day_names through _build_fields(), validates against allowed_values in save_config().
  New-keys modal uses dropdowns for allowed_values/day_names fields. Modal descriptions
  switch from esc() to innerHTML so HTML bold renders. Main Config Editor descriptions
  unchanged (still raw HTML tags visible — fixed in v3.4.38).

v3.4.38 — Config Editor main editor fix. Added allowed_values and day_names dropdown
  branches to the Jinja2 template ({% elif field.get('allowed_values') %} etc.).
  Fixed {{ field.desc }} → {{ field.desc | safe }} so HTML bold renders correctly
  in the main editor. Confirmed: Summary Frequency shows dropdown, descriptions show
  bold values.

v3.4.39 — Summary content config card on Notifications page.
  Five new SUMMARY_SHOW_* bool config keys (Alerts section):

WHAT WAS ATTEMPTED BUT FAILED
------------------------------
- v3.4.37 main editor: bold descriptions still showed raw HTML tags. Root cause:
  {{ field.desc }} in Jinja2 auto-escapes HTML. Fixed in v3.4.38 with | safe filter.
- v3.4.39 Send Test Summary: "name 'alerts' is not defined" error. Root cause:
  alerts not imported at module level. Fixed in v3.4.40 with local import.


DECISIONS MADE
--------------
- Release Manager description flows at Build time (zip is the source of truth).
  Reset is non-fatal — if zip already gone, UI resets cleanly.
- MFA_REQUIRED defaults to True (safe default). mfa_enforcement_enabled() reads
  config at runtime so restart takes effect cleanly.
- Summary frequency is system-wide (not per-subscriber). Simple enough for home install.
- save_quiet route added for toggle-style settings that don't need a service restart.
  Currently only used for SUMMARY_SHOW_* keys. Document pattern for future use.
- allowed_values in config_schema triggers dropdown in both modal and main editor.
  day_names triggers a "N — DayName" dropdown. Both patterns reusable for future keys.
- Test summary sends to logged-in admin's subscriber email only (not all subscribers).
- alerts.py import pattern: always use local "import alerts" inside functions in
  webapp.py — it is never imported at module level.
