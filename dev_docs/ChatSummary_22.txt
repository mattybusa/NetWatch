v3.3.56 — Backup email delivery fix (see Bug_BackupEmailDelivery.txt):
  Both backup.sh and backup_db.sh read config.ALERT_EMAIL which does not exist.
  The correct key is config.ALERT_TO. The 2>/dev/null suppressed the
  AttributeError silently; scripts fell back to GMAIL_USER, sending emails to
  the NetWatch account itself. One-line fix in each script.

v3.3.57–v3.3.59 — Session idle timeout not enforcing logout

WHAT WAS ATTEMPTED BUT FAILED
------------------------------
- v3.3.57 (part 1 of timeout fix): Removed @auth.login_required from
  /api/session/status and /api/session/ping so they return JSON instead of
  redirecting on expiry. Correct fix but not sufficient alone — timeout still
  not enforcing.

- v3.3.58 (part 2 of timeout fix): Added /api/session/status and
  /api/session/ping to _POLLING_PATHS so before_request wouldn't reset
  last_activity on the 30-second status check. Still not sufficient — every
  other auto-refreshing page was also resetting last_activity.

- v3.3.59 (part 3, working fix): Removed update_last_activity() from
  before_request entirely. The blocklist approach was fundamentally fragile —
  any polling endpoint not in the list resets the idle clock. last_activity is
  now updated only by /api/session/ping (real user events). Confirmed working.

- First test of v3.3.59 appeared to fail — user returned after 40 minutes still
  logged in. Root cause: old session cookie from before the fix was still active.
  Logging out and back in gave a clean session and the timeout worked correctly.


DECISIONS MADE
--------------
- Backup email fix: both updater.py upload paths (backup_file and save_dev_doc)
  already create dated backups per v3.3.51/52. No additional changes needed.

- Session idle timeout: last_activity updated ONLY via /api/session/ping (real
  user events: mousedown, keydown, touchstart, scroll, wheel). The before_request
  approach is permanently abandoned — any new polling endpoint would silently
  break idle timeout without a corresponding _POLLING_PATHS update.

- PERMANENT_SESSION_LIFETIME set to 2 days as a hard ceiling on browser cookie
  lifetime. Idle timeout logic handles the normal case; this prevents cookies
  persisting for Flask's 31-day default.

- After installing a session timeout fix, users must log out and back in to get
  a clean session. Old cookies carry stale last_activity timestamps.

- Decrypt-on-Pi rejected for backup restore: private key must not live on the Pi.
  Restore flow is: decrypt locally on Windows → upload decrypted file → restore
  via UI. Backlog item updated accordingly.

- Package storage + push and three related items (distributable installer, update
  package installer, git strategy) noted as a cohesive distribution story — design
  together, not independently.

- utcnow() deprecation warning in backup scripts: harmless, noted, deferred.

- End-of-session context doc updated in closeout package.
