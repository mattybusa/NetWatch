- v3.3.41 — GPG keyring fix for backup scripts:
  GPG refuses to use a keyring owned by a different user. Both backup scripts
  were calling gpg without --homedir, causing it to look in /nonexistent/.gnupg.
  Fix: created dedicated GPG keyring for netwatch-svc at
  /home/mboynton/netwatch/.gnupg, imported the public key into it, updated both
  scripts to use --homedir /home/mboynton/netwatch/.gnupg.
  Manual steps: mkdir + chmod 700 + chown + gpg export + gpg import as netwatch-svc.
  Smoke tested: both backup scripts complete cleanly as netwatch-svc.
  tar "socket ignored" warnings are harmless (gpg-agent sockets, gunicorn.ctl).

- All three cron jobs fully verified working end to end as netwatch-svc.
  First time any of them have successfully run since netwatch-svc migration (v3.3.25).


WHAT WAS ATTEMPTED BUT FAILED
------------------------------
- v3.3.41 first build pointed --homedir at /home/mboynton/.gnupg (mboynton's keyring).
  GPG rejected it: "unsafe ownership" -- will not use a keyring owned by a different
  user regardless of permissions. Required a dedicated keyring for netwatch-svc.

  (full output via crontab wrapper). Both approaches used together: logger for
  structured summary queryable in Loki; systemd-cat for full apt output in journald.

- GPG keyring location: /home/mboynton/netwatch/.gnupg -- inside the netwatch
  directory, owned by netwatch-svc. Keeps all netwatch-svc-owned files together.

- BACKUP_DIR hardcoded to /home/mboynton/backups rather than derived from script
  location. Backups belong to the user's home, not the netwatch directory.

- tar socket warnings: accepted as harmless. Not worth adding exclusion flags for
