WHAT WAS ATTEMPTED BUT FAILED
------------------------------
Email PNG icon (v3.2.53, v3.2.59): Base64 PNG embedded in email HTML. Not rendered
by Gmail (Gmail strips data:image/png base64 src attributes). Confirmed this is a
Gmail policy — emoji is the correct solution.

Emoji icon ghost (v3.2.53–v3.2.57): Several packages updated the emoji but email
showed no change. Root cause: alerts.py is imported by both services; packages were
only restarting the monitor. Web service had old cached module. Resolved by always
restarting both services.

HTTP-hang bug (pre-gunicorn): Flask dev server hangs on HTTP requests to HTTPS port
and blocks all requests temporarily. Workaround: always use https://. Fixed by
gunicorn migration.

Self-introduced regression — Password reset (v3.2.75–v3.2.77): While adding SMS send
to password reset, an incorrect read of the code concluded that password_hash was
still being overwritten (it was not — v3.2.70 had already fixed it). This "fix"
removed the dual-hash write. Then v3.2.76 was needed because must_change_pass was not
cleared on real-password login. Then v3.2.77 was needed because the temp password
stopped working (reset_token_hash now not checked in authenticate()). ~1 hour wasted.
User pushed back correctly each time. v3.2.77 is functionally equivalent to v3.2.70.

Self-introduced regression — Metrics datetime queries (v3.2.85): While investigating
the 24H data gap, concluded (incorrectly) that datetime('now') in SQLite was timezone-
affected. Rewrote all datetime queries in database.py to use Python timedelta cutoffs.
This was unnecessary and wrong — the original queries were correct. Reverted in
v3.2.86 alongside the bucket anchoring fix.

Metrics data gap anchoring (v3.2.82–v3.2.83): Two failed attempts to fix the apparent
24H data gap by expanding startMs to match oldest data. Both were wrong approaches.
Correct fix (v3.2.86): always anchor startMs to now - requestedHours so the x-axis
spans the full window; gaps in data show as breaks in the line.

Google Voice SMS gateway: Researched — @txt.voice.google.com is not a static inbound
gateway. Addresses are per-conversation with random keys. Not added to carrier list.
